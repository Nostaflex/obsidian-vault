# La mitose cognitive produit N notes indépendantes depuis 1 source multi-concepts — chaque note doit passer le test de la fonction pure

Tags: #pkm #second-brain #cognitive-mitosis #atomicity #concept #note-taking

## Essentiel
Quand une source contient plusieurs idées distinctes, la couper en N notes séparées (mitose) est obligatoire. Le critère d'atomicité : chaque note résultante doit être compréhensible sans les autres — comme une fonction pure sans état externe.

## Détail
La mitose cognitive répond à l'échec du « fourre-tout » : une note qui contient 3 concepts croisés ne peut pas être liée sélectivement, ne peut pas être promue indépendamment, ne peut pas être réfutée partiellement. Elle bloque la recombinabilité. Le test opérationnel est le test de la fonction pure : peut-on comprendre cette note sans lire les autres notes de la même source ? Si non, la note est trop couplée et doit être redécoupée. Dans le workflow avec Claude : toute source de plus de 2 concepts distincts déclenche automatiquement une mitose. Claude produit N drafts plutôt qu'1 synthèse. La règle seuil de 2 est délibérément basse — mieux vaut trop de mitoses que pas assez. Une note excessivement atomique se regroupe via les `## Liens` ; une note trop grosse ne peut pas être atomisée après coup sans perdre ses cross-références.
