# Les agents LLM doivent publier leurs actions dans un log partagé avant exécution pour assurer la fiabilité

Tags: #cloud #agent-autonome #fiabilité-production #log-partagé #validation-action

## Essentiel
Chaque agent expose ses intentions dans une source commune avant agir. Cela rend visibles les mutations d'état, permettant inspection et intervention externe. Fondamental pour debugger des systèmes multi-agents.

## Détail
Au lieu de laisser les agents muter directement l'environnement, LogAct interpose un log partagé. L'agent déclare son action future ; des votants externes (découplés, pluggables) évaluent si l'action est sûre ; seulement après approbation l'action s'exécute. En cas de crash, l'historique permet rejouer l'état cohérent.
