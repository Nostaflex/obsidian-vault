# Le partage inter-workflow de composants stables réduit l'empreinte mémoire cluster

Tags: #cloud #memory-efficiency #model-pooling #shared-resources

## Essentiel
Certains modèles d'un pipeline (encodeurs texte, VAE) changent rarement. En les isolant comme services partagés, plusieurs workflows concurrents consultent la même instance mémoire. Économie exponentielle si paralléllation haute.

## Détail
Une stratégie naïve lance une copie de l'encodeur CLIP pour chaque requête utilisateur. LegoDiffusion les fusionne : N requêtes utilisent 1 instance CLIP avec file d'attente. Extension naturelle : un pool de modèles stateless réutilisables, découvert via profiling (quels modèles sont appelés par >1 workflow). Gestion des dépendances : garantir cohérence entrée/sortie entre services.
